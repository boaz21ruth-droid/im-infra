群文件库 — 存储方案

文件从哪里来？从哪里查？

A

聚合已有文件消息（最轻）

群文件页 = 搜索历史消息中 contentType == file(43) 的消息，列表展示。上传就是在聊天里发文件，群文件页只是过滤视图。

优点

  • 零后端改动
  • 文件随消息撤回自然删除

缺点

  • 文件历史受消息保留期限制
  • 无法单独删文件
B

自定义消息 type 914（推荐）

发文件走 createCustomMessage(type=914)，payload 含 url/name/size/uploader；群文件页搜索 type=914 消息并渲染列表。文件本身走 OpenIM 自带 OSS 上传。

优点

  • 零后端改动
  • 文件气泡与普通文件消息区分，可单独处理
  • 扩展字段灵活（缩略图、到期时间等）

缺点

  • 依赖 OpenIM 消息历史（搜索 API）
C

im-business 后端存储文件记录

新增 group_files 表，im-business 提供上传/列表/删除 API；Flutter 调用 REST，不走消息通道。

优点

  • 独立管理，可分页/权限控制

缺点

  • 需要改后端 + 数据库迁移
  • 文件不出现在聊天记录里
